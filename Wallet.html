Apa Itu Bitcoin Wallet?

Abstrak: Bitcoin Wallet

Bitcoin wallet adalah alat esensial dalam ekosistem Bitcoin yang memungkinkan pengguna menyimpan, mengelola, dan bertransaksi dengan Bitcoin melalui pengelolaan kunci kriptografis, yaitu private key dan public key. Berbeda dari dompet fisik, wallet tidak menyimpan Bitcoin secara langsung, melainkan menyimpan kunci untuk mengakses dana di blockchain, buku besar digital terdesentralisasi. Wallet tersedia dalam berbagai jenis, termasuk hot wallet (terhubung internet, praktis untuk transaksi harian), cold wallet (offline, aman untuk penyimpanan jangka panjang), dan paper wallet (fisik, untuk keamanan maksimal). Fitur utama meliputi pembuatan alamat Bitcoin (seperti Bech32 untuk SegWit), pengelolaan Unspent Transaction Outputs (UTXO), dan penandatanganan transaksi dengan algoritma ECDSA. Wallet modern mendukung teknologi seperti Hierarchical Deterministic (HD) Wallet, Lightning Network untuk transaksi cepat, dan multi-signature untuk keamanan tambahan. Keamanan wallet bergantung pada perlindungan seed phrase, pencegahan phishing, dan penggunaan Hardware Security Module (HSM) pada cold wallet. Dengan perkembangan seperti SegWit dan Taproot, Bitcoin wallet terus berkembang untuk meningkatkan efisiensi, privasi, dan aksesibilitas, menjadikannya pintu gerbang utama untuk interaksi dengan Bitcoin secara desentralisasi.

Jenis-Jenis Bitcoin Wallet

Bitcoin wallet dapat diklasifikasikan berdasarkan cara penyimpanan kunci dan konektivitasnya ke internet. Berikut adalah jenis-jenis utama:

Hot Wallet

  • Definisi: Wallet yang terhubung ke internet, seperti aplikasi mobile, desktop, atau berbasis web.
  • Detail Teknis: Hot wallet menyimpan private key di perangkat online (misalnya, ponsel atau komputer) dengan enkripsi seperti AES-256. Contohnya termasuk Trust Wallet (mobile), Electrum (desktop), atau Blockchain.com (web).
  • Kelebihan: Praktis untuk transaksi sehari-hari, cepat, dan mudah digunakan. Mendukung fitur seperti SPV (Simplified Payment Verification) untuk operasi ringan tanpa mengunduh blockchain penuh.
  • Kekurangan: Rentan terhadap malware, phishing, atau serangan siber karena private key tersimpan di perangkat online.
  • Contoh Praktis: Kamu menggunakan Trust Wallet di ponsel untuk mengirim 0,01 BTC ke teman. Transaksi dilakukan langsung dari aplikasi dengan biaya otomatis berdasarkan mempool.

Cold Wallet

  • Definisi: Wallet yang menyimpan private key secara offline, seperti hardware wallet (Ledger, Trezor) atau paper wallet.
  • Detail Teknis: Cold wallet menggunakan Hardware Security Module (HSM), seperti chip secure element, untuk menjaga private key tetap aman. Transaksi ditandatangani offline, mencegah paparan ke internet.
  • Kelebihan: Sangat aman untuk penyimpanan jangka panjang, kebal terhadap serangan online.
  • Kekurangan: Kurang praktis untuk transaksi harian karena memerlukan perangkat fisik atau langkah tambahan (misalnya, memindai QR code untuk paper wallet).
  • Contoh Praktis: Kamu menyimpan 2 BTC di Ledger Nano X. Untuk mengirim dana, kamu menghubungkan Ledger ke komputer, memverifikasi transaksi di layar perangkat, dan menandatanganinya di secure element.

Custodial vs Non-Custodial Wallet

  • Custodial Wallet: Private key disimpan oleh pihak ketiga (misalnya, exchange seperti Binance atau Coinbase). Pengguna tidak memiliki kontrol penuh atas dana.
  • Kelebihan: Mudah digunakan, cocok untuk pemula, sering terintegrasi dengan platform perdagangan.
  • Kekurangan: Risiko tinggi jika penyedia diretas atau bangkrut. Contoh: Wallet di exchange.
  • Non-Custodial Wallet: Pengguna memiliki kendali penuh atas private key (misalnya, Electrum, Ledger).
  • Kelebihan: Lebih aman dan sesuai dengan prinsip desentralisasi Bitcoin.
  • Kekurangan: Tanggung jawab penuh ada pada pengguna untuk mengamankan private key dan seed phrase.
  • Contoh Praktis: Kamu menggunakan Electrum (non-custodial) untuk mengelola Bitcoinmu sendiri, sedangkan temanmu menyimpan Bitcoin di wallet Binance (custodial).

Paper Wallet

  • Definisi: Bentuk cold wallet di mana private key dan alamat Bitcoin dicetak di kertas, sering kali dengan QR code.
  • Detail Teknis: Dibuat offline menggunakan alat seperti bitaddress.org untuk menghindari risiko digital. Private key diimpor ke wallet digital saat ingin digunakan.
  • Kelebihan: Sangat aman untuk penyimpanan jangka panjang jika disimpan di tempat fisik yang aman.
  • Kekurangan: Rentan terhadap kerusakan fisik (kertas robek, tinta luntur) dan kurang praktis untuk transaksi.
  • Contoh Praktis: Kamu membuat paper wallet untuk menyimpan 1 BTC di brankas. Saat ingin menjual, kamu mengimpor private key ke Electrum.

Fungsi Utama Bitcoin Wallet

Bitcoin wallet memiliki beberapa fungsi inti yang memungkinkan pengguna berinteraksi dengan blockchain:

Menyimpan Kunci Kriptografis

  • Wallet menyimpan private key dan public key dengan aman, baik di perangkat online (hot wallet) atau offline (cold wallet). HD Wallet menggunakan derivation path untuk menghasilkan banyak alamat dari satu seed phrase, meningkatkan efisiensi dan privasi.

Menerima Bitcoin

  • Wallet menghasilkan Bitcoin address untuk menerima dana. Alamat ini bisa dibagikan dalam bentuk teks atau QR code. Wallet modern mendukung format seperti Bech32 untuk transaksi SegWit yang lebih murah.
  • Contoh: Kamu membagikan alamat bc1q... ke teman untuk menerima 0,1 BTC. Walletmu otomatis mendeteksi transaksi masuk melalui blockchain explorer atau koneksi ke node.

Mengirim Bitcoin

  • Masukkan alamat penerima, jumlah, dan pilih transaction fee. Wallet akan memilih UTXO, menandatangani transaksi, dan menyiarkannya ke jaringan melalui node.
  • Detail Teknis: Wallet menghitung biaya berdasarkan ukuran transaksi (byte) dan kondisi mempool. Fitur seperti RBF (Replace-by-Fee) memungkinkan penyesuaian biaya jika transaksi tertunda.
  • Contoh: Kamu mengirim 0,2 BTC ke alamat teman. Wallet memilih UTXO 0,3 BTC, mengirim 0,2 BTC ke penerima, dan mengembalikan 0,099 BTC (setelah fee) ke change address.

Memantau Saldo dan Transaksi

  • Wallet menampilkan saldo dengan menjumlahkan UTXO yang terkait dengan alamatmu dan memantau status transaksi (pending atau confirmed) melalui SPV atau koneksi ke node.
  • Contoh: Walletmu menunjukkan saldo 0,7 BTC dan status transaksi dengan TxID di blockchain explorer seperti mempool.space.

Backup dan Pemulihan

  • Wallet menyediakan seed phrase atau private key untuk backup, memungkinkan pemulihan dana di perangkat baru. Mnemonic phrase (seed phrase) adalah cara paling umum untuk HD wallet.
  • Keamanan: Seed phrase harus disimpan offline untuk mencegah pencurian. Wallet sering memperingatkan pengguna untuk tidak menyimpan seed phrase secara digital.

Selalu Jaga Keamanan Bitcoin Wallet

Keamanan adalah aspek kritis dari Bitcoin wallet karena Bitcoin bersifat irreversible (transaksi tidak bisa dibatalkan) dan desentralisasi (tidak ada pihak ketiga untuk membantu pemulihan). Berikut adalah poin-poin keamanan utama:

Melindungi Private Key dan Seed Phrase

  • Private key dan seed phrase adalah kunci akses ke dana. Simpan offline di tempat aman (misalnya, brankas atau pelat logam) dan hindari penyimpanan digital seperti screenshot atau cloud.
  • Contoh Risiko: Malware seperti keylogger bisa mencuri seed phrase jika disimpan di ponsel.

Menghindari Phishing dan Malware

  • Gunakan wallet dari sumber tepercaya (misalnya, situs resmi Ledger atau Electrum). Verifikasi URL untuk menghindari situs phishing.
  • Contoh: Situs palsu yang menyerupai blockchain.com bisa mencuri seed phrase jika kamu login.

Menggunakan Hardware Wallet untuk Penyimpanan Besar

  • Hardware wallet (seperti Ledger, Trezor) menggunakan HSM (Hardware Security Module) untuk menjaga private key tetap offline, bahkan saat terhubung ke komputer yang terinfeksi.
  • Contoh: Kamu menyimpan 5 BTC di Ledger untuk investasi jangka panjang, aman dari serangan online.

Mengelola Dust Attack

  • Penyerang bisa mengirim dust (jumlah kecil Bitcoin) untuk melacak alamat. Gunakan coin control untuk menghindari menggabungkan dust dengan UTXO lain.
  • Contoh: Kamu menerima 200 satoshi dari sumber tidak dikenal dan menandainya sebagai “do not spend” di wallet.

Menghindari Address Reuse

  • Menggunakan alamat yang sama berulang kali memudahkan pelacakan. Wallet modern menghasilkan alamat baru untuk setiap transaksi.
  • Contoh: Walletmu otomatis membuat alamat baru setelah menerima 0,1 BTC untuk transaksi berikutnya.

Memantau Konfirmasi

  • Tunggu 1-6 konfirmasi untuk memastikan transaksi aman, terutama untuk jumlah besar. Gunakan blockchain explorer untuk memeriksa TxID.
  • Contoh: Kamu menunggu 3 konfirmasi (~30 menit) sebelum menganggap transaksi 1 BTC ke exchange aman.

Komponen Inti Bitcoin Wallet

Private Key

  • Definisi: Private key adalah kunci rahasia berupa angka 256-bit yang membuktikan kepemilikan Bitcoin dan digunakan untuk menandatangani transaksi, memungkinkan pengguna untuk "membelanjakan" dana di blockchain.
  • Penjelasan Teknis:
    • Private key adalah bilangan acak yang dihasilkan menggunakan CSPRNG (Cryptographically Secure Pseudo-Random Number Generator) untuk memastikan keamanan kriptografis. Rentangnya berada antara 1 dan n-1 (dengan n sebagai order kurva secp256k1, yaitu FFFFFFFF FFFFFFFF FFFFFFFF FFFFFFFE BAAEDCE6 AF48A03B BFD25E8C D0364141).
    • Private key digunakan dalam algoritma ECDSA (Elliptic Curve Digital Signature Algorithm) untuk menghasilkan tanda tangan digital, yang memverifikasi bahwa transaksi dibuat oleh pemilik tanpa mengungkapkan kunci itu sendiri.
    • Dalam wallet, private key disimpan dalam format Wallet Import Format (WIF), yang menambahkan version byte (0x80 untuk mainnet) dan checksum (4 byte dari double SHA-256), lalu dienkode ke Base58Check. Contoh WIF: 5J3mBbAH58CpQ3Y5RNJpUKPE62SQ5tfcvU2JpbnkeyhfsYB1Jcn (uncompressed) atau Kz... (compressed).
    • Dalam Hierarchical Deterministic (HD) Wallet, private key turunan dihasilkan dari master seed menggunakan derivation path (misalnya, m/44'/0'/0'/0/0), sesuai standar BIP-32.
  • Fungsi dalam Wallet:
    • Private key adalah inti keamanan wallet, memungkinkan pengguna untuk menandatangani transaksi, seperti mengirim Bitcoin ke alamat lain.
    • Wallet menyimpan private key dengan cara yang aman: di secure element (pada hardware wallet seperti Ledger) untuk cold wallet, atau dienkripsi (misalnya, dengan AES-256) pada hot wallet seperti Electrum atau Trust Wallet.
    • Wallet juga memungkinkan impor private key (via WIF) untuk mengakses dana dari wallet lama atau paper wallet.
  • Keamanan:
    • Private key adalah rahasia utama. Jika dicuri, penyerang dapat mengendalikan semua Bitcoin terkait. Jika hilang, dana tidak dapat diakses.
    • Hot Wallet: Private key rentan terhadap malware, keylogger, atau phishing jika perangkat terhubung ke internet.
    • Cold Wallet: Private key disimpan di Hardware Security Module (HSM) atau kertas (paper wallet), sehingga aman dari serangan online, tetapi rentan terhadap pencurian fisik atau kerusakan.
    • Praktik terbaik: Simpan private key offline, gunakan PIN atau kata sandi untuk enkripsi, dan hindari berbagi dalam bentuk digital (misalnya, email atau screenshot).
  • Contoh Praktis:
    • Kamu menggunakan Ledger Nano X untuk menandatangani transaksi 0,1 BTC. Private key disimpan di HSM dan tidak pernah keluar dari perangkat, bahkan saat terhubung ke komputer. Wallet menampilkan detail transaksi di layar perangkat untuk verifikasi manual.
    • Jika kamu memiliki paper wallet, kamu mengimpor private key dalam format WIF ke Electrum untuk membelanjakan dana, lalu membuat wallet baru untuk keamanan.

Public Key dan Bitcoin Address

  • Definisi: Public key adalah kunci kriptografis yang dihasilkan dari private key, digunakan untuk membuat Bitcoin address, yaitu pengenal unik yang dibagikan untuk menerima Bitcoin. Bitcoin address adalah representasi yang lebih ringkas dari public key, dioptimalkan untuk penggunaan praktis.
  • Penjelasan Teknis:
    • Public key dihasilkan melalui perkalian titik pada kurva eliptik secp256k1: Public Key = private_key × G, di mana G adalah titik generator tetap. Public key bisa dalam format uncompressed (65 byte, dimulai dengan 0x04) atau compressed (33 byte, dimulai dengan 0x02 atau 0x03 tergantung paritas y).
    • Bitcoin address dibuat dari public key melalui proses:
      • Hitung SHA-256 dari public key.
      • Hitung RIPEMD-160 (RACE Integrity Primitives Evaluation Message Digest 160-bit) dari hasil SHA-256 untuk mendapatkan public key hash (20 byte).
      • Tambahkan version byte (misalnya, 0x00 untuk P2PKH, 0x05 untuk P2SH).
      • Tambahkan checksum (4 byte dari double SHA-256).
      • Enkode ke Base58Check (untuk P2PKH/P2SH) atau Bech32 (untuk SegWit).
    • Jenis alamat:
      • P2PKH (Pay-to-Public-Key-Hash): Dimulai dengan “1”, contoh: 1A1zP1eP5QGefi2DMPTfTL5SLmv7DivfNa.
      • P2SH (Pay-to-Script-Hash): Dimulai dengan “3”, sering untuk multisig, contoh: 3J98t1WpEZ73CNmQviecrnyiWrnqRhWNLy.
      • Bech32 (SegWit): Dimulai dengan “bc1”, contoh: bc1qar0srrr7xfkvy5l643lydnw9re59gtzzwf5mdq, lebih efisien dan mendukung SegWit.
    • Dalam HD Wallet, wallet menghasilkan banyak alamat dari extended public key (xpub) menggunakan derivation path, meningkatkan privasi dengan mencegah address reuse.
  • Fungsi dalam Wallet:
    • Wallet menghasilkan Bitcoin address untuk menerima dana, biasanya dalam bentuk teks atau QR code untuk memudahkan berbagi.
    • Wallet otomatis membuat alamat baru untuk setiap transaksi (khususnya di HD wallet) untuk mencegah pelacakan melalui analisis rantai.
    • Wallet juga mengelola change address, yaitu alamat baru yang dibuat untuk menerima sisa dana (kembalian) setelah transaksi.
  • Keamanan:
    • Public key dan Bitcoin address aman untuk dibagikan karena tidak mengungkapkan private key. Namun, address reuse harus dihindari karena memudahkan pelacakan aktivitas.
    • Wallet harus memverifikasi alamat penerima untuk mencegah kesalahan pengetikan atau serangan man-in-the-middle (misalnya, malware yang mengganti alamat).
    • Alamat Bech32 lebih tahan terhadap kesalahan input karena menggunakan checksum BCH, tetapi tidak semua platform mendukungnya.
  • Contoh Praktis:
    • Kamu membuka Trust Wallet dan menghasilkan alamat Bech32 (bc1q...) untuk menerima 0,2 BTC dari teman. Wallet menampilkan QR code, dan temanmu memindainya untuk mengirim dana.
    • Saat mengirim 0,1 BTC dari UTXO 0,15 BTC, wallet membuat change address baru untuk mengembalikan 0,049 BTC (setelah fee) ke walletmu.

Seed Phrase (Mnemonic Phrase)

  • Definisi: Seed phrase, atau mnemonic phrase, adalah daftar kata (12-24) yang bertindak sebagai cadangan (backup) untuk memulihkan semua private key dan alamat dalam HD Wallet, memungkinkan pemulihan dana jika perangkat hilang atau rusak.
  • Penjelasan Teknis:
    • Seed phrase dihasilkan sesuai standar BIP-39 dari entropi acak (128-256 bit) menggunakan CSPRNG. Entropi ini di-hash dengan SHA-256 untuk membuat checksum, lalu dibagi menjadi 11-bit chunks yang dipetakan ke daftar 2048 kata BIP-39.
    • Contoh seed phrase (12 kata): witch collapse practice feed shame open despair creek road again ice least.
    • Seed phrase diolah dengan PBKDF2 (Password-Based Key Derivation Function 2) (dengan 2048 iterasi HMAC-SHA512) untuk menghasilkan master seed (512 bit), yang menjadi dasar pembuatan master private key dan chain code sesuai BIP-32.
    • Dari master seed, wallet menghasilkan private key turunan melalui derivation path (misalnya, m/44'/0'/0'/0/0 untuk alamat Bitcoin pertama).
    • Seed phrase bersifat deterministik, artinya wallet yang sama selalu dihasilkan dari seed yang sama, memungkinkan pemulihan di wallet lain yang mendukung BIP-39/BIP-32.
  • Fungsi dalam Wallet:
    • Seed phrase adalah mekanisme utama untuk backup dan pemulihan wallet. Saat membuat wallet baru, pengguna diminta menuliskan seed phrase dan menyimpannya dengan aman.
    • Wallet seperti Trust Wallet, Electrum, atau Ledger menampilkan seed phrase saat inisialisasi dan memungkinkan impor seed phrase untuk memulihkan dana di perangkat baru.
    • Beberapa wallet mendukung passphrase tambahan (opsional) untuk keamanan ekstra, yang digabungkan dengan seed phrase untuk menghasilkan master seed berbeda.
  • Keamanan:
    • Seed phrase adalah titik kritis keamanan. Jika dicuri, penyerang dapat mengakses semua dana di wallet. Jika hilang, dana tidak dapat dipulihkan.
    • Praktik terbaik: Simpan seed phrase offline (misalnya, di kertas, pelat logam, atau brankas), hindari penyimpanan digital (screenshot, cloud), dan jangan bagikan.
    • Risiko: Malware, phishing, atau kamera pengintai bisa mencuri seed phrase. Wallet harus memperingatkan pengguna untuk tidak menyimpan seed phrase di perangkat online.
    • Hardware wallet seperti Ledger menyimpan seed phrase di HSM, yang hanya ditampilkan sekali saat inisialisasi, mengurangi risiko paparan.
  • Contoh Praktis:
    • Saat membuat wallet di Trust Wallet, kamu menuliskan seed phrase 12 kata di kertas dan menyimpannya di brankas. Jika ponselmu hilang, kamu memasukkan seed phrase ke wallet baru untuk memulihkan dana.
    • Kamu menambahkan passphrase tambahan di Ledger untuk membuat wallet “tersembunyi,” meningkatkan keamanan jika seed phrase utama bocor.

Unspent Transaction Output (UTXO)

  • Definisi: UTXO adalah jumlah Bitcoin yang tersedia untuk dibelanjakan di wallet, berasal dari transaksi sebelumnya yang belum digunakan sebagai input untuk transaksi baru.
  • Penjelasan Teknis:
    • Bitcoin menggunakan model UTXO, bukan saldo bank. Setiap transaksi menghasilkan output (jumlah Bitcoin yang dikirim ke alamat tertentu), yang menjadi UTXO hingga digunakan sebagai input untuk transaksi berikutnya.
    • Setiap UTXO diidentifikasi oleh Transaction ID (TxID) (hash 64 karakter dari transaksi) dan output index (vout). Contoh: UTXO dari TxID f4184fc596403b9d... dengan vout 0 bernilai 0,5 BTC.
    • Wallet melacak semua UTXO terkait alamatnya melalui koneksi ke node atau SPV (Simplified Payment Verification), lalu menjumlahkannya untuk menampilkan saldo total.
    • Saat membuat transaksi, wallet memilih UTXO yang cukup untuk menutupi jumlah yang dikirim ditambah transaction fee. Sisa dana dikembalikan ke change address yang dibuat otomatis.
    • Contoh: Kamu memiliki dua UTXO (0,3 BTC dan 0,2 BTC, total 0,5 BTC). Untuk mengirim 0,4 BTC, wallet menggunakan kedua UTXO, mengirim 0,4 BTC ke penerima, dan mengembalikan sisa (misalnya, 0,099 BTC setelah fee) ke change address.
    • Fitur coin control memungkinkan pengguna memilih UTXO secara manual untuk mengoptimalkan biaya atau privasi.
  • Fungsi dalam Wallet:
    • Wallet menghitung saldo dengan menjumlahkan nilai semua UTXO yang terkait dengan alamatnya.
    • Saat mengirim Bitcoin, wallet memilih UTXO secara otomatis (atau manual via coin control) dan membuat transaksi dengan input (UTXO yang digunakan), output (alamat penerima dan change address), dan fee.
    • Wallet menampilkan riwayat transaksi berdasarkan UTXO yang masuk dan keluar, sering kali terhubung ke blockchain explorer untuk memeriksa status konfirmasi.
  • Keamanan:
    • UTXO hanya bisa dibelanjakan oleh pemilik private key yang sesuai, memastikan keamanan dana. Wallet harus memverifikasi bahwa UTXO belum dibelanjakan ganda (double-spending) melalui koneksi ke node.
    • Dust attack adalah risiko di mana penyerang mengirim UTXO kecil untuk melacak aktivitas. Wallet harus mendeteksi dan mengelola dust (UTXO <546 satoshi) dengan hati-hati, misalnya dengan fitur coin control.
    • Menggabungkan banyak UTXO dalam satu transaksi bisa mengurangi privasi jika menghubungkan alamat dari aktivitas berbeda. Wallet modern menghindari ini dengan manajemen alamat yang baik.
  • Contoh Praktis:
    • Walletmu menunjukkan saldo 0,7 BTC dari tiga UTXO: 0,4 BTC, 0,2 BTC, dan 0,1 BTC. Saat mengirim 0,5 BTC, wallet memilih UTXO 0,4 BTC dan 0,2 BTC, mengirim 0,5 BTC ke penerima, dan mengembalikan 0,099 BTC (setelah fee) ke change address baru.
    • Kamu menggunakan coin control di Electrum untuk menghindari UTXO kecil (dust) dari transaksi tidak dikenal, mencegah pelacakan.

Proses Teknis Cold Wallet untuk Transaksi Bitcoin

Cold wallet, seperti hardware wallet (Ledger Nano X, Trezor Model T), dirancang untuk menjaga private key tetap offline dan aman di dalam chip khusus (biasanya secure element). Saat melakukan transaksi, cold wallet menangani penandatanganan transaksi secara offline, sehingga private key tidak pernah terekspos ke perangkat yang terhubung internet. Berikut adalah penjelasan teknis nya:

Colok Cold Wallet ke Komputer via USB

Apa yang terjadi secara teknis:

  • Hardware wallet (misalnya Ledger Nano X) dihubungkan ke komputer melalui port USB atau, untuk beberapa model, via Bluetooth (tapi kita fokus ke USB untuk contoh ini).
  • Wallet berkomunikasi dengan komputer menggunakan protokol seperti USB HID (Human Interface Device) atau WebUSB untuk browser-based apps. Protokol ini memungkinkan pertukaran data yang aman antara hardware wallet dan software di komputer (misalnya, Ledger Live atau Electrum).
  • Hardware wallet memiliki secure element (chip khusus seperti STMicroelectronics ST33 pada Ledger) yang menyimpan private key. Chip ini dirancang tahan terhadap serangan fisik atau digital, seperti side-channel attacks.
  • Komputer mengenali wallet sebagai perangkat eksternal, tapi tidak bisa mengakses private key. Semua operasi sensitif (seperti penandatanganan) dilakukan di dalam chip.
  • Pengguna biasanya harus memasukkan PIN di hardware wallet untuk membuka kunci perangkat. PIN ini disimpan di secure element dan melindungi akses ke private key.
  • Jika wallet mendukung BIP-39 (Bitcoin Improvement Proposal 39), private key dihasilkan dari seed phrase (12-24 kata) yang dibuat saat inisialisasi wallet. Seed ini disimpan di secure element dan tidak pernah keluar.

Di Komputer, Buat Transaksi Secara Online

Apa yang terjadi secara teknis:

  • Kamu membuka software wallet di komputer, seperti Ledger Live, Electrum, atau aplikasi berbasis browser yang kompatibel dengan hardware wallet (misalnya, Metamask dengan ekstensi Ledger).
  • Software ini terhubung ke jaringan Bitcoin (via node atau API seperti Blockstream) untuk mengambil informasi blockchain, seperti:
    • Unspent Transaction Outputs (UTXO): Daftar Bitcoin yang tersedia di alamatmu untuk dibelanjakan.
    • Biaya transaksi (fee): Estimasi biaya jaringan berdasarkan kondisi mempool (antrean transaksi Bitcoin).
  • Kamu memasukkan detail transaksi, seperti:
    • Alamat penerima (contoh: bc1qar0srrr7xfkvy5l643lydnw9re59gtzzwf5mdq).
    • Jumlah Bitcoin yang akan dikirim (misalnya, 0,01 BTC).
    • Biaya transaksi (misalnya, 10 satoshi/byte).
  • Software wallet membuat transaksi mentah (raw transaction) dalam format data yang sesuai dengan protokol Bitcoin. Transaksi mentah ini berisi:
    • Inputs: UTXO yang akan dibelanjakan (referensi ke transaksi sebelumnya).
    • Outputs: Alamat penerima dan jumlah Bitcoin, serta alamat kembalian (jika ada).
    • Fee: Selisih antara input dan output untuk membayar miner.

Contoh transaksi mentah (dalam hex): 0100000001abcdef...00000000

Transaksi ini belum ditandatangani, jadi belum valid untuk dikirim ke jaringan Bitcoin.

Data Transaksi Dikirim ke Cold Wallet via USB

Apa yang terjadi secara teknis:

  • Software wallet mengirimkan transaksi mentah ke hardware wallet melalui koneksi USB menggunakan protokol seperti APDU (Application Protocol Data Unit), yang standar untuk komunikasi dengan secure element.
  • Data yang dikirim berisi:
    • Hash dari transaksi (untuk ditandatangani).
    • Detail input dan output (untuk ditampilkan di layar wallet).
  • Hardware wallet menampilkan detail transaksi di layar perangkat (misalnya, jumlah BTC dan alamat penerima) untuk verifikasi manual. Ini mencegah serangan man-in-the-middle, di mana malware di komputer memalsukan alamat penerima.
  • Pengguna memverifikasi detail di layar hardware wallet (misalnya, “Send 0,01 BTC to bc1q...”) dan menekan tombol fisik untuk menyetujui.

Cold Wallet Menandatangani Transaksi

Apa yang terjadi secara teknis:

  • Setelah pengguna menyetujui, hardware wallet menggunakan private key (disimpan di secure element) untuk menandatangani transaksi menggunakan algoritma ECDSA (Elliptic Curve Digital Signature Algorithm) dengan kurva secp256k1 (kurva eliptik standar yang digunakan Bitcoin).

Proses penandatanganan:

  • Hardware wallet menghitung hash dari transaksi mentah menggunakan SHA-256 (sesuai protokol Bitcoin, hash ini disebut Transaction ID atau TxID).
  • Private key digunakan untuk menghasilkan tanda tangan digital (digital signature), yang terdiri dari dua nilai: r dan s.
  • Tanda tangan ini memverifikasi bahwa transaksi dibuat oleh pemilik private key tanpa mengungkapkan private key itu sendiri.

Contoh tanda tangan (dalam format DER): 3045022100a1b2c3...0220d4e5f6...

Keamanan penting: Private key tidak pernah meninggalkan secure element. Semua operasi kriptografi dilakukan di dalam chip, sehingga komputer (meskipun terinfeksi malware) tidak bisa mengaksesnya.

Hardware wallet juga mendukung BIP-32 (Bitcoin Improvement Proposal 32) (HD wallet), sehingga bisa menggunakan private key turunan (derived key) untuk setiap transaksi, meningkatkan privasi.

Hasil Data + Tanda Tangan Dikirim ke Komputer via USB

Apa yang terjadi secara teknis:

  • Hardware wallet mengirimkan transaksi yang sudah ditandatangani (signed transaction) kembali ke software wallet di komputer melalui USB.
  • Transaksi yang sudah ditandatangani berisi:
    • Transaksi mentah asli.
    • Tanda tangan digital (r, s) untuk setiap input.
  • Software wallet memverifikasi bahwa tanda tangan valid (opsional, untuk memastikan integritas).

Contoh transaksi yang sudah ditandatangani (dalam hex): 0100000001abcdef...3045022100a1b2c3...00000000

Transaksi Disiarkan ke Jaringan Bitcoin

Apa yang terjadi secara teknis:

  • Software wallet mengirim transaksi yang sudah ditandatangani ke jaringan Bitcoin melalui node (misalnya, node publik atau node pribadi jika kamu menjalankan Bitcoin Core).
  • Protokol Bitcoin menggunakan P2P (Peer-to-Peer) network untuk menyebarkan transaksi ke miner.
  • Miner memverifikasi transaksi:
    • Memastikan tanda tangan valid menggunakan public key (diambil dari alamat pengirim).
    • Memeriksa bahwa UTXO yang digunakan belum dibelanjakan (no double-spending).
    • Memastikan fee cukup untuk diproses.
  • Jika valid, transaksi masuk ke mempool (memory pool, kumpulan transaksi yang belum dikonfirmasi) dan menunggu konfirmasi (biasanya dimasukkan ke blok oleh miner dalam 10-60 menit, tergantung fee).
  • Setelah dikonfirmasi (biasanya 1-6 konfirmasi), transaksi dianggap final, dan Bitcoin sampai ke penerima.

Transaksi Berhasil

Pengguna bisa melihat transaksi di blockchain explorer (misalnya, mempool.space) menggunakan TxID (Transaction ID).

Contoh TxID: f4184fc596403b9d6b8e7f2e...

Hardware wallet kembali ke mode offline setelah transaksi selesai, menjaga private key tetap aman.

Proses Teknis Hot Wallet untuk Transaksi Bitcoin

Hot wallet menyimpan private key di perangkat yang terhubung internet (ponsel, komputer, atau server untuk custodial wallet). Prosesnya lebih sederhana dibanding cold wallet karena semuanya dilakukan di satu perangkat. Berikut alurnya:

Buka Hot Wallet di Perangkat

Apa yang terjadi:

  • Kamu buka aplikasi hot wallet (misalnya Trust Wallet di ponsel atau Electrum di PC).
  • Wallet memuat private key dari penyimpanan lokal (file terenkripsi atau database aplikasi) setelah kamu masukkan kata sandi atau PIN.
  • Private key dihasilkan dari seed phrase (BIP-39) saat setup awal, disimpan di perangkat dengan enkripsi (misalnya AES-256 (Advanced Encryption Standard 256-bit)).

Buat Transaksi Secara Online

Apa yang terjadi:

  • Wallet terhubung ke node Bitcoin (via API atau node publik) untuk ambil data seperti UTXO (Unspent Transaction Outputs) (dana yang bisa dibelanjakan) dan estimasi fee.
  • Kamu masukkan:
    • Alamat penerima (contoh: bc1qar0srrr7xfkvy5l643lydnw9re59gtzzwf5mdq).
    • Jumlah (misalnya 0,01 BTC).
    • Fee (misalnya 10 satoshi/byte).
  • Wallet membuat transaksi mentah (raw transaction) berisi inputs (UTXO), outputs (alamat tujuan + kembalian), dan fee.

Contoh transaksi mentah (hex): 0100000001abcdef....

Tanda Tangan Transaksi

Apa yang terjadi:

  • Wallet menggunakan private key di perangkat untuk menandatangani transaksi dengan algoritma ECDSA (Elliptic Curve Digital Signature Algorithm) (kurva secp256k1).

Proses:

  • Hitung hash transaksi (SHA-256).
  • Tanda tangan (r, s) dihasilkan: s = k^(-1) * (z + r * dA) mod n, di mana z adalah hash transaksi, dA adalah private key.
  • Tanda tangan ditambahkan ke transaksi mentah.

Contoh tanda tangan (DER): 3045022100a1b2c3....

Catatan: Semua ini dilakukan di perangkat yang online, jadi rentan jika perangkat terinfeksi malware.

Siaran ke Jaringan Bitcoin

Apa yang terjadi:

  • Wallet mengirim transaksi yang sudah ditandatangani ke jaringan Bitcoin via node.
  • Miner memverifikasi tanda tangan dan UTXO (Unspent Transaction Outputs), lalu masukkan transaksi ke mempool (memory pool).
  • Setelah dikonfirmasi (1-6 blok), transaksi selesai. Kamu bisa cek TxID (Transaction ID) di explorer seperti mempool.space.

Perbedaan Utama Hot Wallet dengan Cold Wallet

Lokasi Penandatanganan

  • Hot Wallet:
    • Proses penandatanganan dilakukan di perangkat online.
    • Rentan terhadap malware atau serangan siber.
  • Cold Wallet:
    • Penandatanganan terjadi di perangkat offline (secure element).
    • Private key tidak pernah terpapar ke internet.

Tingkat Keamanan

  • Hot Wallet:
    • Lebih berisiko terhadap hacking, phishing, keylogger.
    • Jika custodial, private key disimpan oleh pihak ketiga (exchange/wallet provider).
  • Cold Wallet:
    • Private key selalu offline.
    • Kebal terhadap serangan remote (kecuali physical theft + PIN diketahui).

Kemudahan Penggunaan

  • Hot Wallet:
    • Lebih cepat & praktis.
    • Semua proses (buat transaksi, tanda tangan, broadcast) dilakukan di satu perangkat.
  • Cold Wallet:
    • Membutuhkan perangkat tambahan (hardware wallet).
    • Proses transaksi lebih lama karena verifikasi manual.

Apa Itu Multi-Signature (Multisig)?

Multi-signature atau multisig adalah fitur keamanan di Bitcoin (dan kripto lain) yang membutuhkan lebih dari satu private key untuk menandatangani dan mengotorisasi transaksi. Berbeda dengan wallet biasa (single-signature) yang cuma butuh satu private key, multisig memungkinkan pengaturan seperti "M-of-N", di mana M tanda tangan dari N private key diperlukan untuk memvalidasi transaksi.

Contoh: Dalam pengaturan 2-of-3, ada 3 private key, tapi transaksi hanya valid jika 2 di antaranya menandatangani.

Analogi: Seperti brankas bank yang butuh dua kunci dari dua orang untuk dibuka.

Multisig meningkatkan keamanan dan fleksibilitas, cocok untuk skenario seperti dompet bersama, escrow, atau perusahaan yang ingin kontrol terdistribusi.

Proses Pembuatan Bitcoin Wallet dari Nol (Private Key Only)

Pembuatan wallet Bitcoin melibatkan kriptografi untuk menghasilkan private key, public key, dan Bitcoin address. Berikut langkah-langkah teknisnya:

Menghasilkan Random Number (Entropi)

Apa yang terjadi:

  • Wallet dimulai dengan menghasilkan angka acak 256-bit (32 byte) untuk menjadi private key.
  • Angka ini harus benar-benar acak dan aman secara kriptografi.
  • Digunakan CSPRNG (Cryptographically Secure Pseudo-Random Number Generator), seperti:
    • /dev/urandom di Linux
    • os.urandom di Python
    • Fungsi TRNG (True Random Number Generator) di hardware wallet.

Keamanan kritis:

  • Jika random number lemah (misalnya pakai RNG biasa), private key bisa ditebak, dan Bitcoin hilang.

Contoh output:

Random number (hex): E9873D79C6D87DC0FB6A5778633389F4453213303DA61F20BD67FC233AA33262

Membuat Private Key

Apa yang terjadi:

  • Random number 256-bit langsung dijadikan private key.
  • Dalam Bitcoin, private key harus berada dalam rentang:
    • 1 ≤ private key ≤ n-1
    • (di mana n = FFFFFFFF FFFFFFFF FFFFFFFF FFFFFFFE BAAEDCE6 AF48A03B BFD25E8C D0364141, order kurva secp256k1)

Format Penyimpanan (Wallet Import Format - WIF):

  1. Tambahkan version byte (mainnet: 0x80)
  2. Hitung checksum:
    • checksum = 4 byte pertama dari SHA-256(SHA-256(private_key + version))
  3. Encode ke Base58Check (versi Base58 dengan checksum).

Contoh:

  • Raw private key (hex): 
    E9873D79C6D87DC0FB6A5778633389F4453213303DA61F20BD67FC233AA33262
  • WIF (Wallet Import Format): 
    5J3mBbAH58CpQ3Y5RNJpUKPE62SQ5tfcvU2JpbnkeyhfsYB1Jcn

Menghasilkan Public Key

Apa yang terjadi:

  • Private key (dA) dikonversi menjadi public key menggunakan ECDSA (Elliptic Curve Digital Signature Algorithm) dengan kurva secp256k1.

Proses Matematis:

  1. Perkalian titik elliptic curve: 
    Public Key = dA × G
    • dA = private key (bilangan integer)
    • G = generator point (titik tetap pada kurva secp256k1)
  2. Hasilnya berupa koordinat (x, y) di kurva eliptik.

Format Public Key:

  1. Uncompressed (65 byte):
    • Diawali 0x04 + koordinat x + koordinat y
    • Contoh: 
      04x_coordinatey_coordinate...
  2. Compressed (33 byte):
    • Hanya menyimpan x + parity byte (0x02 jika y genap, 0x03 jika y ganjil)
    • Contoh: 
      02x_coordinate... (jika y genap)
03x_coordinate... (jika y ganjil)

Contoh Nyata:

  • Private key (hex): 
    E9873D79C6D87DC0FB6A5778633389F4453213303DA61F20BD67FC233AA33262
  • Public key (compressed, 33 byte): 
    02F8BDE4F3...

Membuat Bitcoin Address

Dari public key, wallet menghasilkan Bitcoin address (format P2PKH (Pay-to-Public-Key-Hash)) melalui langkah-langkah berikut:

Proses Pembuatan Address:
  1. Hitung SHA-256 (Secure Hash Algorithm 256-bit) dari public key.
  2. Hitung RIPEMD-160 (RACE Integrity Primitives Evaluation Message Digest 160-bit) dari hasil SHA-256 → menghasilkan public key hash (20 byte).
  3. Tambahkan version byte:
    • Mainnet (P2PKH): 0x00
  4. Hitung checksum:
    • checksum = 4 byte pertama dari SHA-256(SHA-256(version + pubkey_hash))
  5. Encode ke Base58Check (versi yang lebih pendek dari Base64, menghindari karakter ambigu).
Contoh Nyata:
  • Public key (compressed, 33 byte): 
    02F8BDE4F3...
  • Public key hash (RIPEMD-160, 20 byte): 
    62E907B15CBF27...
  • Bitcoin address (Base58Check): 
    1A1zP1eP5QGefi2DMPTfTL5SLmv7DivfNa

Proses Pembuatan Bitcoin Wallet dengan Seed Phrase

Generate Random Number (Entropi)

Hasilkan entropi acak (128-256 bit) menggunakan CSPRNG (Cryptographically Secure Pseudo-Random Number Generator) (contoh: os.urandom).

  • Contoh 128-bit entropi (untuk 12 kata): 
    a1b2c3d4e5f6... (16 byte)

Create Seed Phrase (BIP-39)

  • Entropi di-hash dengan SHA-256 (Secure Hash Algorithm 256-bit) untuk menghasilkan checksum (beberapa bit terakhir).
  • Entropi + checksum dibagi menjadi 11-bit chunks, lalu dipetakan ke daftar kata BIP-39 (Bitcoin Improvement Proposal 39) (2048 kata).
  • Contoh 12-kata seed phrase: 
    witch collapse practice feed shame open despair creek road again ice least

Generate Master Seed

  • Seed phrase + optional passphrase diolah menggunakan PBKDF2 (Password-Based Key Derivation Function 2) (HMAC-SHA512, 2048 iterasi) untuk menghasilkan master seed (512 bit).
  • Contoh master seed (hex): 
    4e8c18...

Create Private Key (BIP-32)

  • Dari master seed, buat master private key dan chain code menggunakan HMAC-SHA512 (Hash-based Message Authentication Code - Secure Hash Algorithm 512-bit).
  • Derive private key turunan (child keys) menggunakan BIP-32 (Bitcoin Improvement Proposal 32) path (contoh: m/44'/0'/0'/0/0 untuk Bitcoin).
  • Contoh private key turunan: 
    e8f32e723dec...

Generate Public Key

  • Gunakan ECDSA (Elliptic Curve Digital Signature Algorithm) secp256k1: 
    Public Key = private_key * G
  • Contoh public key (compressed): 
    02f8bde4...

Create Bitcoin Address

  • Public key → SHA-256 (Secure Hash Algorithm 256-bit) → RIPEMD-160 (RACE Integrity Primitives Evaluation Message Digest 160-bit) → tambahkan version byte (0x00 untuk P2PKH) → Base58Check.
  • Contoh Bitcoin address: 
    1A1zP1eP5QGefi2D...
